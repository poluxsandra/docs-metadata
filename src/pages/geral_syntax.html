Sintaxe

Aqui está uma resumo da sintaxe ao colocar metadados em elementos <head> usando o padrão AGLS Metadata

Namespace

Um namespace é um arquivo legível por máquina que fornece definições do esquema de metadados. Identificar o namespace de um termo de metadados é necessário para que os metadados sejam processáveis.

Os prefixos de namespace são usados ​​nos nomes das propriedades para indicar o agrupamento lógico e a identificação única de um conjunto de termos de metadados dos quais a propriedade é tomadas:

  • DCTERMS para Dublin Core
  • AGLSTERMS para AGLS

As identidades dos namespaces, usando o elemento <link>, têm o seguinte padrão: 


<link rel="schema.PREFIX" href="namespaceURI">

A codificação do namespace para as versões atuais do Dublin Core e AGLS são: 


<link rel="schema.DCTERMS" href="http://purl.org/dc/terms/">

<link rel="schema.AGLSTERMS" href="http://www.agls.gov.au/agls/terms/">

Implementações de legado usando elementos DC e AGLS devem fazer referência ao legado namespaces: 


<link rel="schema.DC" href="http://purl.org/dc/elements/1.1/">

<link rel="schema.AGLS" href="http://www.agls.gov.au/agls/1.2/">

Propriedades

As propriedades são codificadas usando os atributos de nome e conteúdo do elemento <meta>.

Usar os seguintes formulários são recomendados: 


<meta name="DCTERMS.property" content="valor">

<meta name="AGLSTERMS.property" content="valor">

Por exemplo: 


<meta name="DCTERMS.date" content="2007-07-18">

<meta name="AGLSTERMS.function" content="Educação escolar">

Os caracteres especiais no "valor" do atributo content podem ser codificados como uma referência de entidade de caractere HTML. Para exemplo, uma letra e acentuada: 


<meta name="DCTERMS.creator" content="Da Costa, José"[& eacute;]>

Encoding schemes

Quando um esquema é especificado, o namespace do Vocabulary Encoding Scheme ou Syntax Encoding Scheme também deve ser especificado, por exemplo, "DCTERMS.URI" ou "AGLSTERMS.AGIFT".

O valor deve ser codificado no conteúdo de acordo com esse esquema, incluindo o uso de quaisquer caracteres de pontuação. Use os seguintes formulários: 


<meta name="DCTERMS.property" scheme="DCTERMS.Scheme" content="valor">

<meta name="AGLSTERMS.property" scheme="AGLSTERMS.Scheme" content="valor">

Por exemplo: 


<meta name="DCTERMS.date" scheme="DCTERMS.ISO8601" content="2007-07-18">

<meta name="DCTERMS.audience" scheme="AGLSTERMS.agls-audience" content="pessoas com deficiência"[& ecirc;]>

<meta name="AGLSTERMS.function" scheme="AGLSTERMS.AGIFT" content="Escola de Educação"[& ccedil;][& atilde;]>

Idioma do Valor

Também é recomendável especificar o idioma dos valores quando estes forem livres para o desenvolvedor. Por exemplo: 


<meta name="DCTERMS.title" lang="pt-BR" content="Diretório de organizações"[& oacute;][& ccedil;][& otilde;]>

Exemplo

Esta é uma página de exemplo: (contém mais do que o número mínimo de propriedades necessárias) 


<!DOCTYPE html>
<html>
<head>
<title>Services to Government</title>
<link rel="schema.DCTERMS" href="http://purl.org/dc/terms/">
<link rel="schema.AGLSTERMS" href="http://www.agls.gov.au/agls/terms/">
<meta name="DCTERMS.identifier" scheme="DCTERMS.URI"
content="http://www.naa.gov.au/records-management/">
<meta name="DCTERMS.creator" scheme="AGLSTERMS.AglsAgent"
content="corporateName = National Archives of Australia; address = Queen
Victoria Terrace, Parkes, ACT; contact = + 61 2 6212 3600">
<meta name="DCTERMS.publisher" scheme ="AGLSTERMS.AglsAgent" content="
corporateName = Arquivos Nacionais da Austrália; address = Queen
Victoria Terrace, Parkes, ACT; contact = + 61 2 6212 3600">
<meta name="DCTERMS.rights" scheme="DCTERMS.URI"
content="http://www.naa.gov.au/info/copyright.aspx">
<meta name="DCTERMS.title" content="Records Management">
<meta name="DCTERMS.subject" scheme="AGLSTERMS.APAIS" content="archives; information management; public administration">
<meta name="DCTERMS.description" lang="en-AU" content="This page provides access to information about records and recordkeeping in the Commonwealth, including references to standards, guidelines and advice.">
<meta name="DCTERMS.language" scheme="DCTERMS.RFC4646" content="en-AU">
<meta name="AGLSTERMS.jurisdiction" scheme="AGLSTERMS.AglsJuri"
content="Commonwealth of Australia">
<meta name="AGLSTERMS.function" scheme="AGLSTERMS.AGIFT"
content="recordkeeping standards">
<meta name="DCTERMS.created" scheme="DCTERMS.ISO8601" content="1998-08-27">
<meta name="DCTERMS.modified" scheme="DCTERMS.ISO8601" content="2008-01-29">
<meta name="AGLSTERMS.aggregationLevel" content="collection">
<meta name="AGLSTERMS.category" content="service">
<meta name="AGLSTERMS.documentType" scheme="AGLSTERMS.agls-document"
content="guidelines">
<meta name="DCTERMS.format" scheme="DCTERMS.IMT" content="text/html">
</head>
</body>
<p>Page content</p>
</body>
</html>
