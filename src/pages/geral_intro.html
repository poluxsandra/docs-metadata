Introdução

Metadados de descoberta de recursos são informações em um formato estruturado que descreve um recurso ou uma coleção de recursos. Um registro de metadados consiste em um conjunto de propriedades que descrevem um recurso.

1) Definições de Termos


Nome Definição
Class (Classe) Um grupo contendo membros que possuem atributos, comportamentos, relacionamentos ou semânticas em comum; é uma espécie de categoria.
DCMI Dublin Core Metadata Initiative, a agência de manutenção do Dublin Core.
Modelo Abstrato DCMI Um conjunto de componentes e construções usadas nos metadados Dublin Core, fornecendo uma informação modelo independente de qualquer sintaxe de codificação.
Domain (Domínio) Uma relação entre uma propriedade e uma classe que indica que se a propriedade faz parte de um par propriedade/valor, segue-se que o recurso descrito é uma instância dessa classe.
Literal O valor de uma propriedade de metadados que pode ser um hiperlink (URI) ou um valor de string (texto livre).
Metadata (Metadados) Informações estruturadas e processáveis ​​por máquina que descrevem e/ou permitem encontrar, gerenciar, controlar, compreender ou preservar outras informações ao longo do tempo. Pode ser compreendido como "dados sobre dados".
Namespace Um namespace é um agrupamento lógico de propriedades. Ele define as regras para que as propriedades pertençam ao grupo.
Property (Propriedade) Um aspecto, característica, atributo ou relação específica usado para descrever um recurso. Dublin Core e os termos de metadados AGLS são propriedades.
Sub-property (Subpropriedade) A instância específica de uma propriedade, que esclarece o significado da propriedade.
Range (Alcance) Uma relação entre uma propriedade e uma classe que indica que se a propriedade faz parte de um par propriedade/valor, segue-se que o valor é uma instância dessa classe.
Resource (Recurso)

Qualquer coisa que tenha uma identidade.

Os exemplos incluem um documento eletrônico (pdf, html, docx), uma imagem, um serviço e uma coleção de outros recursos.

Nem todos os recursos podem ser recuperados pela web. Humanos, corporações, objetos físicos e documentos eletrônicos em pendrives também são recursos.

Service (Serviço) Um serviço existe onde há um relacionamento estabelecido entre uma função de negócios de uma organização e as necessidades identificadas de um cliente individual ou de um grupo de clientes.
Syntax Encoding Scheme (Esquema de Codificação de Sintaxe) Indica que o valor é uma string formatada de acordo com uma notação formal ou padrão definido externamente. (Portanto tem relação com a forma do recurso)
Vocabulary Encoding Scheme (Esquema de Codificação de Vocabulário) Indica que o valor é um termo de um um conjunto, finito e predefinido por convenção, de palavras. (Portanto tem relação com o sentido do recurso)
Value (Valor) O conteúdo de uma propriedade de metadados que fornece informações sobre uma característica de um recurso.

2) Categorias de Obrigação


No AglsTerms/DCTerms são 4 categorias: (Todas as propriedades dos outros namespaces dessa documentação são opcionais)


Categoria Definição
Obrigatório Essas propriedades devem estar presentes em todos os registros de metadados.
Condicional Essas propriedades devem estar presentes em determinadas circunstâncias.
Recomendado Pode haver razões válidas em circunstâncias específicas para não incluir essas propriedades, mas todas as implicações devem ser compreendidas e cuidadosamente avaliadas.
Opcional Essas propriedades são verdadeiramente opcionais

3) Namespace


Um namespace é um arquivo legível por máquina que fornece definições do esquema de metadados. Ao ler o termo que foi especificado a máquina irá questionar de onde veio aquele dado (a resposta é o namespace).

Identificar o namespace de um termo de metadados é necessário para que esse metadado seja processado corretamente.

O namespace é espeficificado por um prefixo antes das propriedades e também pelo elemento <link> dentro do elemento <head>.

A declarações dos namespaces no elemento <link> podem ser das seguintes maneiras:

  • Para DCTerms: 

    
<link rel="schema.DCTERMS" href="http://purl.org/dc/terms/">

  • Para AGLSTerms: 

    
<link rel="schema.AGLSTERMS" href="http://www.agls.gov.au/agls/terms/">

    Para AgentTerms: 

    
<link rel="schema.AGENTTERMS" href="http://www.agls.gov.au/agls/agentterms/">

  • Para AvailTerms: 

    
<link rel="schema.AVAILTERMS" href="http://www.agls.gov.au/agls/AVAILTERMS/">

  • Para AdminTerms: 

    
<link rel="schema.ADMINTERMS" href="http://www.agls.gov.au/agls/adminterms/">

  • Para Legacy DCTerms: 

    
<link rel="schema.DC" href="http://purl.org/dc/elements/1.1/">

  • Para Legacy AGLSTerms: 

    
<link rel="schema.AGLS" href="http://www.agls.gov.au/agls/1.2/">
